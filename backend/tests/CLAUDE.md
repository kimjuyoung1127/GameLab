# Tests — 테스트

## 실행 방법

```bash
cd backend
python -m pytest tests/ -v
```

## 파일 목록

| 파일 | 설명 |
|---|---|
| `test_analysis_harness.py` | 분석 파이프라인 테스트 (11개) |

## test_analysis_harness.py — 테스트 구성

| 범주 | 테스트 내용 |
|---|---|
| 스텝 단위 | 각 파이프라인 스텝의 입출력 검증 |
| 통합 | 전체 파이프라인 end-to-end 실행 |
| 회귀 | 기존 동작 변경 방지 검증 |

## fixtures/ 디렉터리

| 폴더 | 설명 |
|---|---|
| `audio/` | 테스트용 WAV 파일 |
| `expected/` | 기대 출력 결과 |

## 규칙

1. 새 분석 엔진 추가 시 **반드시** 해당 엔진 테스트 추가
2. 새 파이프라인 스텝 추가 시 **반드시** 스텝 단위 테스트 추가
3. 기존 테스트 11개가 모두 통과하는 상태 유지
4. `python -m pytest`로 실행 (모듈 임포트 경로 보장)
